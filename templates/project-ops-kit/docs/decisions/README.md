# Architecture Decision Records (ADRs)

This directory contains Architecture Decision Records for this project.

## What is an ADR?

An ADR captures an important architectural decision along with its context and consequences.

## Template

Use `ADR-TEMPLATE.md` for new decisions.

## Index

| ADR | Title | Status | Date |
|-----|-------|--------|------|
| [ADR-001](ADR-001-example.md) | Example Decision | Accepted | YYYY-MM |

## Status Definitions

- **Proposed** - Under discussion
- **Accepted** - Decision made
- **Deprecated** - No longer applies
- **Superseded** - Replaced by another ADR

## When to Write an ADR

- Choosing a framework or library
- Selecting an architectural pattern
- Making build/deployment decisions
- Defining coding standards
- Any decision that affects multiple team members
