## Screenshot Requirements

### Mac App Store Guidelines

- **Minimum:** 3 screenshots per size
- **Maximum:** 10 screenshots per size
- **Format:** PNG or JPEG (PNG recommended)
- **Color Space:** sRGB or P3
- **Orientation:** Landscape only for macOS
- **Content:** Must show actual app functionality
- **No Alpha:** Screenshots must not have transparency

### General Best Practices

- Show the app in its best light
- Use realistic, relatable data (not "Lorem ipsum")
- Ensure UI elements are sharp and readable
- Avoid sensitive or personal information
- Keep consistent visual style across all screenshots
- Highlight key features in the first 2-3 screenshots

## Screenshot Scenarios

The following 10 scenarios should be captured to showcase CloudSync Ultra's full capabilities.

### Screenshot 1: Dashboard View (First Impression)

**Priority:** Critical - This is the first screenshot users see

**What to Show:**
- Main window with sidebar showing connected remotes
- 3-5 configured cloud providers (Google Drive, Dropbox, OneDrive, etc.)
- Clean, organized interface
- Status indicators showing healthy connections

**Sample Content:**
- Remote named "Personal Google Drive" - Connected
- Remote named "Work Dropbox" - Connected
- Remote named "Backup S3" - Connected
- Show storage usage if visible

**Capture Tips:**
- Use realistic remote names
- Ensure all connections show "connected" status
- Window should fill most of the screenshot

## Screenshot Guidelines

### Content Guidelines

**DO:**
- Use realistic, professional file names
- Show the app performing its core function
- Highlight unique selling points
- Ensure all text is readable
- Use consistent styling across screenshots

**DON'T:**
- Include personal or sensitive data
- Show error states or bugs
- Use placeholder text like "Test123"
- Crop important UI elements
- Show competing product names

### Technical Guidelines

**Resolution:**
- Capture at native resolution (Retina preferred)
- Export at exact required dimensions
- Use PNG for lossless quality
- Verify no compression artifacts

**Window State:**
- App should be active/focused
- No system dialogs overlaying
- Clean Desktop background
- Full window visible (not cropped)

### Design Guidelines

**Consistency:**
- Same window position across shots
- Consistent sample data style
- Matching light/dark mode throughout
- Same sidebar state (expanded/collapsed)

**Framing:**
- Center the main content
- Leave small margin around window
- Consider adding subtle shadow for depth
- Optional: Add device frame or background

## Capture Process

### Pre-Capture Setup

1. **Prepare Sample Data:**
   - Create realistic remote names
   - Populate with sample files/folders
   - Set up scheduled tasks
   - Configure encryption on one remote

2. **Environment:**
   - Clean Desktop background (solid color)
   - Hide Desktop icons
   - Disable notifications temporarily
   - Set display to target resolution

3. **App State:**
   - Launch app fresh
   - Connect all configured remotes
   - Clear any error states
   - Set appropriate light/dark mode

### Capture Commands

```bash
# Capture entire screen
screencapture screenshot.png

# Capture specific window (interactive)
screencapture -w screenshot.png

# Capture with timer (5 seconds)
screencapture -T5 screenshot.png

# Capture to clipboard
screencapture -c
```

### Post-Processing

1. Verify exact dimensions
2. Remove any status bar if captured
3. Add optional background/frame
4. Export as PNG, sRGB color space
5. Name files consistently:
   - `01-dashboard-1280x800.png`
   - `02-file-browser-1280x800.png`
   - etc.
